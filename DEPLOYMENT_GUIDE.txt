🚀 OPTION 1: FIREBASE HOSTING (Recommended + Free Tier)
───────────────────────────────────────────────────────

Best for: Everything integrated, custom domain, SSL certificate included

Steps:
1. Install Firebase CLI:
   npm install -g firebase-tools

2. Login:
   firebase login

3. Initialize hosting in your project folder:
   firebase init hosting

4. When asked:
   - Public directory: . (current directory)
   - Configure as single-page app: No
   - Set up automatic builds: No

5. Deploy:
   firebase deploy --only hosting

6. Your site will be live at: https://your-project-id.web.app

Free tier includes:
✅ 10 GB/month storage
✅ 1 GB/month download
✅ Custom domain support
✅ SSL certificate (automatic)
✅ Global CDN

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🌐 OPTION 2: GITHUB PAGES (Free)
──────────────────────────────────

Best for: Simple, free hosting directly from GitHub

Steps:
1. Create GitHub account (if not already): https://github.com/signup

2. Create new repository:
   - Name: username.github.io (replace username with your GitHub username)
   - Make it Public
   - Click "Create repository"

3. Upload your portfolio files:
   - Clone the repo: git clone https://github.com/your-username/your-username.github.io
   - Copy portfolio.html, firebase-config.js, etc. to the folder
   - Open terminal in that folder
   - Run:
     git add .
     git commit -m "Add portfolio"
     git push

4. Your site will be live at: https://your-username.github.io

Free features:
✅ No cost
✅ Good for portfolio projects
⚠️ Firebase Firestore should still work
⚠️ May need CORS configuration

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

☁️ OPTION 3: NETLIFY (Free with Custom Domain)
──────────────────────────────────────────────

Best for: Easy drag-drop deployment, great for static sites

Steps:
1. Go to: https://netlify.com
2. Sign up with GitHub account
3. Click "Add new site" → "Deploy manually"
4. Drag and drop your portfolio files
5. Done! Your site is live
6. Connect custom domain in Site settings → Domain management

Free tier:
✅ Unlimited builds
✅ Automatic HTTPS
✅ Global CDN
✅ Custom domain support
✅ Form handling (native support!)

🌍 OPTION 5: CUSTOM DOMAIN
────────────────────────────

Buy a domain and connect to any of the above:

Domain registrars:
- Namecheap: https://www.namecheap.com (~$1-3/year)
- GoDaddy: https://www.godaddy.com
- Google Domains: https://domains.google

After buying, update DNS records to point to your hosting provider.

Examples for Firebase Hosting:
- Update A record to Firebase IP
- Update CNAME record as specified by Firebase

❓ QUESTIONS?
───────────

Q: Can I test locally before deploying?
A: Yes! Just open portfolio.html in your browser. It'll work but emails 
   won't send without proper Firebase setup.

Q: Will my contact form work without deploying?
A: Form submissions will be stored locally. To send emails and sync to 
   Firebase, you need proper credentials in firebase-config.js

Q: How do I update my portfolio after deployment?
A: Make changes locally, then redeploy:
   - Firebase: firebase deploy
   - GitHub Pages: git push
   - Netlify: Auto-deploys from GitHub
   - Vercel: Auto-deploys from GitHub

Q: Is my data secure?
A: Yes! Firebase handles security. Firestore rules prevent unauthorized access.
   Emails are sent via Gmail's secure servers.
